# Sheila's Hindi Classes Website

Static HTML/CSS/JS website generated by the 12-agent pipeline.

## Files

- `html/` — one `.html` file per page
- `style.css` — design tokens and styles (do not edit; regenerate via Kavya)
- `shared.js` — navigation, footer, forms (do not edit; regenerate via Vikram)
- `.htaccess` — Apache hosting security config
- `netlify.toml` — Netlify hosting security config
- `_headers` — Netlify alternative security config
- `delivery.md` — full project summary, QA results, and next steps

## Local preview

Serve the project root locally, then open `/html/index.html` in your browser:

```sh
python3 -m http.server 8080
```

## Before going live

1. **Add your photos** — place image files in `assets/images/` using these names:
   - `hero-home.jpg` — hero background (1440×700, Hindi script / cultural scene)
   - `portrait-anuradha.jpg` — teacher portrait (480×580)
   - `student-writing.jpg` — lesson activity photo (400×300)
   - `india-market.jpg` — decorative scene for Why Learn Hindi? page (1000×400)

2. **Connect a form provider** — so contact and booking form submissions reach Anuradha:
   - **Netlify Forms (free):** add `netlify` attribute to both `<form>` tags in `html/contact.html` and `html/book-a-lesson.html`
   - **Formspree (free tier):** set `action="https://formspree.io/f/[your-id]"` on both forms
   - Update `form-action` in your chosen security config file to include the provider domain

3. **Fix two pre-launch accessibility items:**
   - Add `aria-required="true"` to `<select id="booking-level">` in `html/book-a-lesson.html`
   - The `sr-only` class is now defined in style.css (fixed in this version)

## Deployment

Choose one of:

- **Netlify:** Drag and drop this folder to netlify.com/drop. Keep `netlify.toml` OR `_headers` (not both). Remove `.htaccess`. HTTPS is automatic.
- **Apache hosting:** Upload via FTP. Keep `.htaccess`. Remove `netlify.toml` and `_headers`. Enable SSL in your hosting control panel.
- **GitHub Pages, S3, Cloudflare Pages:** Upload the `html/` and `assets/` folders. Configure security headers at the hosting layer using the values in `_headers` as reference, and route `/` to `html/index.html`.

After deployment, verify security headers at https://securityheaders.com
